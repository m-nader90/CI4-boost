# UI, Design & UX Skills Framework

A comprehensive, modernized reference for UI, design, and UX skills organized by discipline area with definitions, techniques, deliverables, tools, and proficiency levels.

## Modern Mindset Principles

Apply these principles across all skill areas:

- Human-centered design - Start with user needs, not business requirements
- Design thinking - Iterate through empathy, definition, ideation, prototyping, testing
- Systems thinking - Consider broader ecosystem impacts and second-order effects
- Lean UX - Minimize waste, validate early, iterate rapidly
- Agile collaboration - Integrate design into sprints, continuous delivery
- Rapid experimentation - Test hypotheses quickly, fail fast, learn faster
- Remote/asynchronous collaboration - Document first, communicate async when possible
- Ethical and privacy-minded design - Consider data use, consent, dark patterns
- Accessibility-by-default - Design for the widest range of users from the start
- Sustainability - Consider digital carbon footprint and longevity
- Data-informed decision making - Combine qualitative insights with quantitative data
- Responsible AI - Use AI-assisted design thoughtfully and transparently

## 1. Core UI Design

### Layout and Composition

Definition: Structuring visual elements to create clear hierarchy, balance, and flow across interfaces.

Key Techniques:
- Grid systems and modular scaling
- Whitespace management for breathing room
- Visual hierarchy through size, color, position
- Responsive breakpoint planning
- Golden ratio and divine proportions
- Content-first layout decisions

Typical Deliverables:
- Layout specifications and spacing systems
- Responsive design documentation
- Component placement guides
- Breakpoint definitions
- Grid templates

Tools: Figma (auto-layout, constraints), Sketch (artboards), Adobe XD (responsive resize)

Proficiency Levels:
- Novice: Applies basic grids, understands whitespace importance, follows templates
- Intermediate: Uses auto-layout fluently, creates responsive designs, handles content variability
- Advanced: Masters complex responsive systems, anticipates edge cases, defines layout standards
- Expert: Innovates new responsive patterns, leads organization-wide layout systems

### Typography

Definition: Selecting, pairing, and implementing typefaces to establish brand voice and ensure readability.

Key Techniques:
- Type scale creation with modular ratios
- Line-height and letter-spacing optimization
- Responsive typography with fluid sizing
- Variable fonts for performance
- WCAG contrast compliance
- Multi-language and script support

Typical Deliverables:
- Type scales and sizing systems
- Font pairing documentation
- Typography tokens for design systems
- Implementation specs for developers
- Responsive typography guidelines

Tools: Figma (text styles), Google Fonts, Adobe Fonts, Fontshare

Proficiency Levels:
- Novice: Selects readable fonts, applies basic sizing hierarchy
- Intermediate: Creates consistent type scales, understands web fonts and loading
- Advanced: Implements variable fonts, optimizes performance, ensures multi-language support
- Expert: Defines brand typography systems, creates custom scales, leads typographic innovation

Key Techniques:

## Quick Reference: Tools by Usage

### Design and Prototyping

Figma - Primary design tool, auto-layout, components, prototyping, collaboration (used for most UI design work)

Sketch - macOS design tool, artboards, symbols (used by teams with established workflows)

Adobe XD - Design and prototype, responsive resize (used in Adobe ecosystem teams)

Framer - Advanced prototyping with code, complex interactions (used for high-fidelity prototypes)

Balsamiq - Low-fidelity wireframes, quick concepts (used for early ideation)

### Motion and Interaction

After Effects - Complex animation, motion graphics (used for polished motion design)

Principle - Mac interaction prototyping (used for Apple platform micro-interactions)

Lottie - Animated icons and illustrations (used for production animation)

### Research and Testing

Maze - Unmoderated usability testing (used for rapid testing)

UserTesting - Moderated and unmoderated research (used for in-depth studies)

Lookback - Live moderated research (used for remote interviews)

Dovetail - Research analysis and synthesis (used for organizing insights)

### Collaboration and Handoff

Miro - Workshops, brainstorming, mapping (used for collaborative sessions)

Mural - Visual collaboration, workshops (used for large group facilitation)

FigJam - Whiteboarding, design jams (used for Figma-centric teams)

Notion - Documentation, planning, collaboration (used for async work)

Jira/Linear - Project management, agile workflows (used for developer coordination)

Zeplin - Design handoff, specs (used by developer-focused teams)

Zeroheight - Design system documentation (used for system publishing)
